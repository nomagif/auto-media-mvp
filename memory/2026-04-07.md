- Investigated LINE webhook on VPS bot.clawfortune.jp. Findings: app via PM2 (`line-webhook`) is healthy on `127.0.0.1:3001`; `GET /health` returns 200, `GET /` returns 404, and `POST /callback` returns 404 directly from Express.
- Public nginx endpoint `https://bot.clawfortune.jp/health` returns 200, but `POST https://bot.clawfortune.jp/callback` returned `200 bot.clawfortune.jp is alive`, indicating nginx is catching the wrong path rather than forwarding to the webhook app.
- `grep` on nginx config showed proxy targets for `http://127.0.0.1:3001/health` and `http://127.0.0.1:3001/line/webhook`; likely path mismatch between LINE Developers webhook URL, nginx, and Express routes. Strong hypothesis: correct webhook path is `/line/webhook`, not `/callback`.
- Confirmed on VPS: `POST /callback` returns 404 from Express, while `POST /line/webhook` returns 200. nginx proxies `/line/webhook` and falls back to `return 200 'bot.clawfortune.jp is alive'` for unmatched paths.
- Fixed by updating the LINE Developers Console webhook URL to `https://bot.clawfortune.jp/line/webhook`. Bot response recovered after the change.
